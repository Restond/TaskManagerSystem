- 项目初始化，完成基础架构
- 集成 Spring Boot & JWT 安全认证
- 实现用户注册、登录（JWT 校验）
- 实现任务的创建、更新、删除、条件查询
- 支持任务状态与优先级筛选、截止日期筛选
- 增加全局异常处理（任务不存在、任务冲突）

- 集成 Lombok 简化代码
- 完成基础接口文档

# [1.0.1] - 2025-11-11
### Added
- 任务状态枚举：`PENDING`, `IN_PROGRESS`, `COMPLETED`。
- 数据验证：任务标题不能为空且长度在1-100之间，优先级必须在1-5之间。
- 全局异常处理：新增验证异常处理，返回详细的字段错误信息。
### Changed
- 将`completed`字段替换为`status`枚举字段。
- 更新查询方法，使用`status`替代`completed`。
- 更新方法以使用枚举状态，并设置新任务的默认状态为`PENDING`。
- 更新状态查询端点，使用枚举类型，并添加了`@Valid`注解以触发验证。

# [1.0.2] - 2025-11-12
### Added
- 新增专用状态更新端点 `PATCH /tasks/{taskId}/status/{status}`。
- 为任务实体添加 `userId` 字段，实现用户与任务的绑定
- 新增任务所有权验证方法，确保用户只能操作自己的任务
- 新增基于当前用户的查询端点，包括按状态、优先级、截止日期过滤
- 集成 Spring Boot Mail，用户登录成功后发送邮件通知
- 新增 `EmailService` 处理邮件发送逻辑
- 用户登录成功后自动发送登录成功邮件到注册邮箱
### Changed
- 所有任务操作现在自动关联当前登录用户，无需手动指定用户ID
- 提取公共的用户验证逻辑，大幅减少代码重复
- 更新所有Repository和Service方法，支持用户隔离的数据访问
- 清理重复和冲突的API端点，优化RESTful接口设计
- 配置 QQ 邮件服务器（SMTP）连接参数
- 设置邮件发送超时和 TLS 加密传输
### Security
- 实现完整的用户任务隔离，防止越权访问
- 任务创建时自动从JWT令牌获取并设置用户ID
- 在删除、更新、查询任务时验证任务所有权
- 所有受保护端点均要求有效的JWT令牌
- 邮件发送失败不影响正常登录流程，通过异常捕获和日志记录确保系统稳定性
### Fixed
- 修正权限验证条件判断逻辑
- 清理重复的API端点映射
- 修复验证异常处理的状态码问题
- 移除不必要的流式操作和重复过滤
### Architecture
- TaskService新增用户相关的查询和验证方法
- TaskRepository添加用户ID与各种条件的组合查询方法
- 统一错误响应格式，提升API一致性
### Logging
- 添加邮件发送过程的详细日志，便于调试和监控
